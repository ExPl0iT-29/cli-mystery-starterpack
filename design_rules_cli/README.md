# CLI Mystery Design Rules

This folder is a reusable study pack for building new command-line mystery games based on the structural lessons in `clmystery`.

It studies the repository from three perspectives:

- game design
- engineering / code architecture
- narrative / writing

And it turns those observations into reusable rules, checklists, and templates.

## Core Conclusion

The main lesson from `clmystery` is that a strong CLI mystery game is not primarily a codebase. It is a carefully authored information architecture where the filesystem, the shell, and the narrative all reinforce each other.
